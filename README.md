# Описание проекта

## Цель проекта

Данный проект представляет собой консольную программу для учета расходов
пользователя за один месяц.

Цель проекта - реализовать программу, которая позволяет добавлять
ежедневные расходы, хранить их по дням и выполнять основные аналитические
операции над введенными данными.

Отдельное внимание в проекте уделено интервальным запросам. Для быстрого
подсчета суммы расходов за период используется массив префиксных сумм.

## Структура проекта

Проект состоит из трех основных файлов:

```text
budget_helper
├── main.py
├── README.md
└── .gitignore
```

Назначение файлов:

- `main.py` - основной файл программы с классами, функциями и консольным
  меню;
- `README.md` - описание проекта, функционала и использованных алгоритмов;
- `.gitignore` - список файлов и папок, которые не должны попадать в
  репозиторий.

## Используемый язык и запуск

Для запуска программы необходимо перейти в папку проекта и выполнить
команду:

```
python3 main.py
```
Также возможен запуск командой:

Далее работа выполняется через консольное меню.

# Функционал программы

Программа предоставляет пользователю следующие возможности:

1. Добавление расхода с указанием дня, суммы и категории.
2. Просмотр списка всех добавленных расходов.
3. Подсчет суммы расходов за выбранный период.
4. Поиск дня с максимальной суммой расходов.
5. Просмотр категорий по общей сумме трат.
6. Отмена последнего добавленного расхода.
7. Вывод расходов через бинарное дерево поиска.
8. Завершение работы программы.

Главное меню программы:

```text
1. Добавить расход
2. Показать все расходы
3. Посчитать сумму расходов за период
4. Найти день с максимальными расходами
5. Показать категории по сумме трат
6. Отменить последний добавленный расход
7. Показать дерево расходов
8. Выйти
```

# Основные данные программы

## Расходы по дням

Суммарные расходы за каждый день месяца хранятся в списке:

```python
daily_expenses = [0] * 32
```

Список имеет длину 32. Индекс 0 не используется, поэтому номер дня месяца
совпадает с индексом списка.

Пример:

```python
daily_expenses[5]
```

Данное значение хранит общую сумму расходов за 5 день месяца.

## Список всех расходов

Все добавленные расходы хранятся в списке:

```python
expenses = []
```

Каждый отдельный расход представлен словарем:

```python
{
    "day": 5,
    "amount": 700,
    "category": "еда"
}
```

Такой формат используется для хранения полной информации о каждом
добавленном расходе.

# Используемые структуры данных и алгоритмы

## Префиксные суммы

Для обработки интервальных запросов используется массив префиксных сумм:

```python
prefix_sums = [0] * 32
```

`prefix_sums[i]` хранит сумму расходов с 1 по i день включительно.

После добавления или отмены расхода массив префиксных сумм пересчитывается.
Сумма расходов за период с дня A по день B вычисляется по формуле:

```python
prefix_sums[B] - prefix_sums[A - 1]
```

Сложность запроса суммы за период: **O(1)**.

## Линейный поиск

Линейный поиск используется для определения дня с максимальной суммой
расходов.

Программа последовательно проходит по списку `daily_expenses` с 1 по 31
день и сравнивает суммы расходов.

Сложность алгоритма: **O(n)**, где `n = 31`.

## Сортировка вставками

Для сортировки категорий по сумме расходов реализована сортировка
вставками.

Категории сортируются по убыванию общей суммы трат. Встроенные функции
`sort()` и `sorted()` для данной операции не используются.

Сложность алгоритма: **O(k²)**, где `k` - количество категорий.

## Стек

Стек используется для отмены последнего добавленного расхода.

При каждом добавлении расход помещается в список:

```python
undo_stack = []
```

При отмене программа извлекает последний добавленный расход с помощью
операции `pop()`.

После отмены выполняются следующие действия:

1. Расход удаляется из общего списка `expenses`.
2. Сумма расходов за соответствующий день уменьшается.
3. Массив `prefix_sums` пересчитывается.
4. Дерево расходов перестраивается.

## Бинарное дерево поиска

Для хранения расходов по дням реализовано бинарное дерево поиска.

Ключ узла дерева - номер дня месяца. Значение узла - список расходов,
относящихся к этому дню.

Структура узла дерева:

```text
day
expenses
left
right
```

Если расход добавляется за день, которого еще нет в дереве, создается
новый узел. Если узел с таким днем уже существует, расход добавляется в
список расходов данного узла.

Для вывода дерева используется симметричный обход:

```text
левое поддерево -> текущий узел -> правое поддерево
```

В результате расходы выводятся по возрастанию дней.

# Обработка пользовательского ввода

В программе предусмотрена проверка вводимых данных:

- день должен быть числом от 1 до 31;
- сумма должна быть положительным числом;
- категория не должна быть пустой строкой;
- при вводе букв вместо числа программа выводит сообщение об ошибке.

Примеры сообщений:

```text
Ошибка: день должен быть числом от 1 до 31.
Ошибка: сумма должна быть положительным числом.
Ошибка: категория не может быть пустой.
```

# Описание основных классов

## ExpenseTreeNode

Класс описывает узел бинарного дерева поиска.

Поля класса:

- `day` - день месяца;
- `expenses` - список расходов за данный день;
- `left` - левое поддерево;
- `right` - правое поддерево.

## ExpenseTree

Класс отвечает за работу с бинарным деревом поиска.

Основные методы:

- `insert(expense)` - добавляет расход в дерево;
- `print_tree()` - выводит дерево по возрастанию дней;
- `clear()` - очищает дерево;
- `rebuild(expenses)` - перестраивает дерево по списку расходов.

## BudgetHelper

Основной класс программы. В нем хранятся данные о расходах и реализованы
основные операции.

Основные методы:

- `add_expense(day, amount, category)` - добавляет новый расход;
- `update_prefix_sums()` - пересчитывает префиксные суммы;
- `get_period_sum(start_day, end_day)` - считает сумму расходов за период;
- `find_max_expense_day()` - ищет день с максимальными расходами;
- `get_category_totals()` - считает суммы по категориям;
- `insertion_sort_categories(category_items)` - сортирует категории;
- `show_categories_by_total()` - выводит категории по сумме трат;
- `undo_last_expense()` - отменяет последний расход;
- `show_all_expenses()` - выводит все расходы;
- `show_tree()` - выводит дерево расходов;
- `rebuild_tree()` - перестраивает дерево.
